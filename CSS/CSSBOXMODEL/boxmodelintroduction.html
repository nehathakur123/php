The Box Model describes how the (visible or non-visible) box around an element is laid out and how margins, paddings and borders behave.

Inline vs. Block elements

  • A block level element will naturally span the entire available width

  • a block level element will automatically push following content to a new line, so by default, two block level elements can't stand next to each other

  • Contrary to the block level element, an inline element does not break the current flow

  • An inline element only takes up the space it needs to render its content and after that, more inline elements can be displayed.

  • Elements are born as either invisible, inline or block level elements

  • An example of inline elements include span tags (the generic inline element), links (anchors), images and form fields like input

  • Common block level elements include div tags (the generic block level element), header tags (h1-h6), paragraphs, lists and tables

  • the width and height properties are ignored for inline elements. Instead, you can use the line-height property to make an inline element smaller or bigger, as the space between each line grows or shrinks.
